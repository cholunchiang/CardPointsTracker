# CardPointsTracker Architecture 🏗️

## Overview

**CardPointsTracker** is a Progressive Web App (PWA) for tracking credit card rewards, points, and benefits. The application is a **single-file PWA** that runs entirely in the browser with no backend, using localStorage for persistence.

This document describes the proposed **modular architecture** that separates concerns while maintaining the ability to build a single-file HTML distribution.

## File Descriptions

### 📄 `src/index.html`

**Purpose:** Contains only HTML structure and DOM elements.

**Content:**
- DOCTYPE, head metadata (charset, viewport, title)
- No inline styles (imported via `<link>` in build)
- No inline scripts (imported via `<script>` in build)
- Semantic HTML5 structure with classes for styling
- IDs for JavaScript targeting
- Form inputs with `oninput`/`onclick` attributes pointing to global functions

**Key Elements:**
- `#app` — main container
- `.top-bar` — header with search
- `.tab-row` — navigation tabs
- `#contentArea` — dynamic content
- `.nav-bar` — bottom navigation
- `.modal-bg` containers for modals (detail, add card, log points)
- `.fab` — floating action button

**Guidelines:**
- Keep HTML structure semantic and clean
- All styling and behavior delegated to CSS and JS
- Use data attributes for configuration when needed
- Avoid inline styles (use classes instead)

- Update seasonal benefits and annual credits

```javascript
let currentCategoryFilter = '';      // Currently active category filter (v1.8)
```

**Category Mapping Constants (v1.8):**
```javascript
const CATEGORY_COLORS = {
  'dining': 'green',
  'travel': 'blue',
  'hotels': 'purple',
  'flights': 'blue',
  'groceries': 'orange',
  'gas': 'red',
  'streaming': 'pink',
  'entertainment': 'pink',
  'drugstores': 'teal',
  'everything': 'gray',
  // ... more mappings
};

const CATEGORY_ALIASES = {
  'dining worldwide': 'Dining',
  'restaurants worldwide': 'Dining',
  'restaurant': 'Dining',
  'everything else': 'Everything',
  'us supermarkets': 'Supermarkets',
  'flights booked directly': 'Flights',
  // ... more normalization aliases
};

const DYNAMIC_COLORS = ['green', 'blue', 'orange', 'purple', 'red', 'pink', 'teal'];
```

**Init Function:**
```javascript
function initApp() {
  cards = loadCards() || DEFAULTS;
  renderAll();
  setupEventListeners(); // optional: delegate event handlers
}
```

**Core Functions:**
- `switchNav(n)` — change current navigation (cards/miles/coupons/summary)
- `setTab(t)` — filter by tab (owner or bank)
- `handleSearch(v)` — filter by search query
- `filtered()` — return filtered card list based on current state

**Category Filter Functions (v1.8):**
- `extractCategory(perk)` — Extract category from perk string (e.g., "3x Dining" → "Dining")
- `normalizeCategory(cat)` — Normalize category using CATEGORY_ALIASES (e.g., "Dining worldwide" → "Dining")
- `getCategoryColor(perk)` — Get color name for a perk's category (uses CATEGORY_COLORS, assigns dynamic colors for unknowns)
- `filterByCategory(category)` — Set currentCategoryFilter and re-render
- `getCardsFilteredByCategory(list)` — Filter cards by category, sort by earning rate (highest first)
- `getEarningRateForCategory(card, category)` — Get earning rate (as number) for card in specific category
- `populateCategoryFilter()` — Populate category dropdown with all available categories from current cards

**Guidelines:**
- Keep state in top-level variables (easy to reason about, debug)
- No complex state management library needed for this app
- All state mutations go through explicit functions (not direct assignment)

#### `storage.js` — Data Persistence & Validation

**Purpose:** Read/write to localStorage, data validation, import/export.

**Public Functions:**

```javascript
function saveCards()
```
- Serialize `cards` array to JSON
- Store in `localStorage.cpCards`
- Called after any card mutation

```javascript
function loadCards()
```
- Read from `localStorage.cpCards`
- Parse JSON safely (try/catch)
- Validate structure (array, required fields)
- Wipe corrupted data automatically
- Return validated array or `null`

```javascript
function exportBackup()
```
- Create JSON with metadata (version, export timestamp)
- Trigger browser download as `.json`
- iOS Safari shows share sheet

```javascript
function importBackup(event)
```
- Read `.json` file from file input
- Parse and validate
- Confirm user wants to replace current data
- Merge and normalize imported cards
- Re-render UI

**Validation Rules:**
- Cards must be an array
- Required fields: `id`, `name`, `bank`, `icon`, `color`, `points`, `owner`, `perks[]`, `redeem[]`, `coupons[]`, `history[]`
- `name` and `bank` must be strings (no Chinese characters)
- `points` must be a number >= 0
- `perks`, `redeem`, `coupons`, `history` must be arrays
- If validation fails, remove corrupted data silently

**Example Validation:**
```javascript
const bad = p.some(c =>
  !c || typeof c !== 'object' ||
  typeof c.name !== 'string' ||
  !Array.isArray(c.perks) ||
  /[\u4e00-\u9fff]/.test(c.name)
);
if (bad) localStorage.removeItem('cpCards');
```

### 🔨 `build/`

Build process combines all modular files into a single HTML distribution.

#### `build.sh`

**Purpose:** Combine src/ files into a single-file PWA for distribution.

**Process:**

```bash
#!/bin/bash

# 1. Read src/index.html
# 2. Inject CSS files in order (base → components → themes → utilities)
# 3. Inject JS files in order (data → logic → initialization)
# 4. Output to dist/CardPointsTracker.html
```

**File Order (critical for dependencies):**

**CSS Order:**
1. `styles/base.css` — Variables and resets
2. `styles/components.css` — Component styles
3. `styles/themes.css` — Color schemes
4. `styles/utilities.css` — Utility classes

**JS Order:**
1. `data/banks.js` — Bank constants (no deps)
2. `data/cards.js` — Card DB (uses BANK_BRANDS)
3. `data/defaults.js` — Sample data (uses CARD_DB)
4. `js/utils.js` — Utilities (no deps)
5. `js/storage.js` — Storage logic (no deps on app)
6. `js/render.js` — Rendering functions (uses utils, card structure)
7. `js/modals.js` — Modal handlers (uses render, storage, utils)
8. `js/app.js` — Main app init (uses all of the above)
9. Inline `initApp()` call

**Example Build Logic:**

```bash
#!/bin/bash

HTML=$(cat src/index.html)

# Inject CSS
CSS=$(cat src/styles/base.css src/styles/components.css src/styles/themes.css src/styles/utilities.css)
CSS_TAG="<style>$CSS</style>"
HTML="${HTML//<\/head>/$CSS_TAG\n<\/head>}"

# Inject JS
JS=$(cat src/data/banks.js src/data/cards.js src/data/defaults.js \
         src/js/utils.js src/js/storage.js src/js/render.js src/js/modals.js \
         src/js/app.js)
JS_TAG="<script>$JS\ninitApp();</script>"
HTML="${HTML//<\/body>/$JS_TAG\n<\/body>}"

echo "$HTML" > dist/CardPointsTracker.html
```

**Build Invocation:**

```bash
cd /path/to/CardPointsTracker
./build/build.sh
# Output: dist/CardPointsTracker.html (ready for deployment)
```

## Build Process Workflow

### Local Development

1. **Edit source files:**
   - Modify `src/styles/*.css` for styling
   - Modify `src/js/*.js` for logic
   - Modify `src/index.html` for structure
   - Update `src/data/*.js` for card/bank info

2. **Test in browser** (during development):
   - Open `src/index.html` in a live server
   - Use individual CSS files (linked via `<link>`)
   - Use individual JS files (loaded via `<script>`)
   - This works because no file imports others

3. **Generate distribution:**
   ```bash
   ./build/build.sh
   ```

4. **Test built version:**
   - Open `dist/CardPointsTracker.html`
   - Verify all styles and features work
   - Test on mobile (Safari, Chrome)

5. **Deploy / Release:**
   - Commit `dist/CardPointsTracker.html` to releases/
   - Share as `.eml` or `.html`
   - Install as PWA

### Development Server (Optional)

Use a simple HTTP server for development:

```bash
cd src
python3 -m http.server 8000
# or
npx http-server
```

Then visit `http://localhost:8000` and use browser dev tools.

## Guidelines for Making Changes Safely

### ✅ Adding a New Credit Card to CARD_DB

1. Open `src/data/cards.js`
2. Add new entry to `CARD_DB` object:
   ```javascript
   'card name in lowercase': {
     bank: 'Bank Name',
     icon: '✈️',
     color: '#XXXXXX',
     perks: ['perk1', 'perk2', ...],
     redeem: ['partner1', 'partner2', ...],
     ratio: '1:1',
     coupons: ['benefit1', 'benefit2', ...]
   }
   ```
3. Rebuild: `./build/build.sh`
4. Test autofill in dev version

### ✅ Adding a New Bank

1. Open `src/data/banks.js`
2. Add to `BANK_BRANDS` object (for branding lookup)
3. Add to `BANK_LIST` array (for picker display)
4. Use official bank colors, verify in dark mode
5. Rebuild and test bank picker

### ✅ Styling Changes

1. Identify which CSS file the component belongs to:
   - Layout/spacing? → `base.css`
   - Component appearance? → `components.css`
   - Colors? → `themes.css`
   - Utilities? → `utilities.css`
2. Make changes
3. Test in dev version (open `src/index.html` with CSS `<link>` tags)
4. Rebuild and test full version
5. Check both light and dark mode

### ✅ Adding a New UI Feature

1. **Structure** — Add HTML to `src/index.html`
2. **Styling** — Add CSS to appropriate file in `src/styles/`
3. **Logic** — Add functions to appropriate file in `src/js/`
4. **Data** — Update `src/data/` if new constants needed
5. **Integration** — Hook up handlers, call from `app.js`
6. Test, rebuild, deploy

### ✅ Fixing a Bug

1. Identify which file(s) are involved
2. Add `console.log()` or debugger statements
3. Open `src/index.html` in dev server
4. Test fix in browser dev tools
5. Remove debug statements
6. Rebuild
7. Test built version
8. Commit changes with clear message

### ⚠️ What NOT to Do

❌ **Don't edit `dist/CardPointsTracker.html` manually** — it's generated.
- Changes will be overwritten by next `./build.sh`

❌ **Don't add circular dependencies in JS files**
- JS order in build: data → utils → storage → render → modals → app
- Respect this order

❌ **Don't put styles inline in HTML**
- All styles in `src/styles/` files
- Use CSS classes in HTML

❌ **Don't use external CDN dependencies** (for now)
- Keep self-contained for offline PWA usage
- No jQuery, Bootstrap, etc. (unless critical)

❌ **Don't modify `src/index.html` structure without updating corresponding CSS/JS**
- Structure and styling are tightly coupled
